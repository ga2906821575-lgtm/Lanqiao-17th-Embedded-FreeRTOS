## 1. 项目概述

本项目为第17届蓝桥杯嵌入式设计与开发省赛真题的参考实现，实现了一个**压力/流量监测与控制系统**。系统通过 ADC 采集模拟电压计算压力值，通过定时器输入捕获测量脉冲频率计算流量值，通过 PWM 输出控制占空比，并通过 LCD 液晶屏显示实时参数，支持按键进行人机交互。

### 1.1 功能需求

| 功能模块 | 说明 |
|---------|------|
| 压力采集 | ADC2 采集 R37 电位器电压，转换为 0~10 BAR 压力值 |
| 流量采集 | TIM3 输入捕获测量 R39 信号频率，转换为 0~40 L/M 流量值 |
| 流量累计 | 基于流量值积分计算累计流量 Q（单位：L） |
| PWM 输出 | TIM2 CH2 输出可调占空比方波，控制执行机构 |
| 工作模式 | 手动模式（按目标占空比输出）/ 自动模式（按目标压力闭环调节） |
| LCD 显示 | 三个界面：主界面、输出设置、参数设置 |
| 按键控制 | 4个按键：界面切换、模式切换/选择、加、减 |
| LED 指示 | 4个 LED 分别指示报警、异常状态、调节状态 |

## 4. 外设资源配置

### 4.1 ADC2 — 压力采集

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 通道 | ADC_CHANNEL_15 | PB15 引脚 |
| 分辨率 | 12位 | 量程 0~4095 |
| 数据对齐 | 右对齐 | 低位有效 |
| 采样时间 | 2.5 个周期 | 最快速度 |
| 触发方式 | 软件触发 | 每次调用 HAL_ADC_Start() 启动 |
| 转换模式 | 单次转换 | 非连续模式 |

**压力计算公式**:
```
r37_volt = 3.3 * ADC_Value / 4095.0

if (r37_volt >= v_offset):
    P = (r37_volt - v_offset) / (3.3 - v_offset) * 10.0   [BAR]
else:
    P = 0
```

其中 `v_offset` 为零点偏移电压，可通过长按 K3 设置。

### 4.2 TIM3 — 频率测量（输入捕获）

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 时钟源 | 内部时钟 | 80MHz |
| 预分频 | 80-1 | 计数频率 = 1MHz |
| 计数周期 | 65535 | 最大捕获时间 65.535ms |
| 通道 | CH1 | PB4 引脚 |
| 捕获极性 | 上升沿触发 | TIM_INPUTCHANNELPOLARITY_RISING |
| 中断 | 使能 | 每次捕获触发中断 |

**频率与流量计算**:
```
fre = 80,000,000 / (80 * capture_value) = 1,000,000 / capture_value [Hz]

if (fre >= 800 && fre <= 8000):
    F = fre / 200.0   [L/M]
elif (fre < 800):
    F = 0
else:  # fre > 8000
    F = 40
```

频率-流量对应关系：
| 频率(Hz) | 流量(L/M) |
|---------|----------|
| < 800 | 0 |
| 800 | 4.0 |
| 2000 | 10.0 |
| 4000 | 20.0 |
| 6000 | 30.0 |
| 8000 | 40.0 |
| > 8000 | 40.0（上限） |

### 4.3 TIM2 — PWM 输出

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 时钟源 | 内部时钟 | 80MHz |
| 预分频 | 80-1 | 计数频率 = 1MHz |
| 计数周期 | 100-1 | PWM 频率 = 10kHz |
| 通道 | CH2 | PA1 引脚 |
| 初始占空比 | 5 | 5% |
| PWM 模式 | PWM1 | 向上计数，CCR 前有效 |
| 极性 | 高电平有效 | OCPOLARITY_HIGH |

**占空比更新公式**:
```c
TIM2->CCR2 = (uint32_t)((D / 100.0) * (TIM2->ARR + 1));
```
其中 `D` 为整数百分比值（5~95），ARR = 99，因此 CCR2 范围约为 5~95。

### 4.4 TIM6 — 流量累计定时器

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 预分频 | 800-1 | 计数频率 = 100kHz |
| 周期 | 10000-1 | 中断周期 = 100ms |
| 中断 | 使能 | 每100ms触发一次 |

**累计流量计算**:
```c
Q += (uint32_t)(F - 0.5);
```
注意：实际积分应该考虑时间间隔，但代码中以100ms为周期直接累加 `(F - 0.5)`，这里 `F` 单位是 L/M（升/分钟），100ms = 1/600 分钟，理论上应乘以 1/600。代码中的简化处理实际上累积速度偏快（约600倍），但从赛题角度通常直接按题目要求处理。

### 4.5 TIM7 — 占空比调节定时器

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 预分频 | 8000-1 | 计数频率 = 10kHz |
| 周期 | 10000-1 | 中断周期 = 1s |
| 中断 | 使能 | 每秒触发一次 |

在 TIM7 中断中完成占空比的渐变调节逻辑：
- **手动模式**: D 逐步趋近 TarD（目标占空比）
- **自动模式**: D 根据压力偏差逐步调节

### 4.6 TIM1 / TIM4 / TIM17 — 自由运行计时器

这三个定时器仅作为**自由运行的计数器**使用，不进行周期性中断，通过读取 `TIMx->CNT` 寄存器实现延时计时。

| 定时器 | 预分频 | 用途 |
|--------|--------|------|
| TIM1 | 8000-1 (10kHz) | LED2 延时点亮计时（2秒） |
| TIM4 | 8000-1 (10kHz) | 按键长按检测计时（2秒） |
| TIM17 | 8000-1 (10kHz) | LED3 延时点亮计时（2秒） |

CNT 溢出时间为 65535 / 10000 = 6.5535 秒，满足 2 秒延时需求。

### 4.7 GPIO — LED 与按键

**LED 控制原理**：
开发板使用 74HC573 锁存器控制 8 个 LED，PC8~PC15 连接 LED 阴极，PD2 为锁存器使能端。

控制时序：
1. 写 PC8~PC15 引脚电平（低电平点亮，高电平熄灭）
2. PD2 置高 → 锁存数据
3. PD2 置低 → 完成锁存

代码实现 (`led_show()`):
```c
HAL_GPIO_WritePin(GPIOC, GPIO_PIN_8 << (led - 1), mode ? GPIO_PIN_RESET : GPIO_PIN_SET);
HAL_GPIO_WritePin(GPIOD, GPIO_PIN_2, GPIO_PIN_SET);
HAL_GPIO_WritePin(GPIOD, GPIO_PIN_2, GPIO_PIN_RESET);
```

## 6. 核心算法与业务逻辑

### 6.1 LCD 界面系统

系统包含三个界面，通过变量 `lcd_face` 控制切换：

| 界面 | `lcd_face` | 功能 |
|------|-----------|------|
| MAIN（主界面） | 0 | 显示实时参数 |
| OUTP（输出设置） | 1 | 设置目标占空比/目标压力 |
| PARA（参数设置） | 2 | 设置报警阈值 |

**界面切换机制**：使用 `lcdX_first` 标志位实现首次进入界面的初始化逻辑。

#### 6.1.1 MAIN 界面

```
┌──────────────────────┐
│       MAIN           │  Line1
│                      │  Line2
│   M=MAN/AUTO         │  Line3
│   P=xx.xBAR          │  Line4
│   F=xx.xL/M          │  Line5
│   Q=xxxL             │  Line6
│   D=xx%              │  Line7
│   V=x.xV             │  Line8
└──────────────────────┘
```

- 首次进入时，将 PARA 界面的临时参数同步到正式参数
- 显示当前工作模式、压力、流量、累计流量、占空比、偏移电压

#### 6.1.2 OUTP 界面

```
┌──────────────────────┐
│       OUTP           │
│                      │
│   TarD=xx%           │
│   TarP=x.xBAR        │
└──────────────────────┘
```

- K2 切换选择 TarD / TarP
- K3/K4 增减选中的参数值

#### 6.1.3 PARA 界面

```
┌──────────────────────┐
│       PARA           │
│                      │
│   FH=xx.xL/M         │
│   FL=xx.xL/M         │
│   PL=x.xBAR          │
│   DH=xx%             │
└──────────────────────┘
```

- K2 循环选择 FH/FL/PL/DH
- K3/K4 增减选中的参数值
- 首次进入时，将 OUTP 界面的临时参数同步到正式参数

### 6.2 按键处理逻辑

| 按键 | 短按功能 | 长按功能（MAIN界面） |
|------|---------|-------------------|
| K1 (PB0) | 切换界面 (0→1→2→0) | 无 |
| K2 (PB1) | 界面内选择切换 | 无 |
| K3 (PB2) | 当前选中项增加 | 长按2秒：设置 v_offset = r37_volt |
| K4 (PA0) | 当前选中项减少 | 长按2秒：清零累计流量 Q = 0 |

**消抖处理**: 按键扫描周期为 50ms，通过检测下降沿触发动作。

**长按检测实现**: 使用 TIM4 自由计数器，按键按下时清零 CNT，持续按下时判断 CNT > 20000（即 2 秒）。

### 6.3 LED 控制逻辑

| LED | 触发条件 | 说明 |
|-----|---------|------|
| LED1 | `r39_fre > 8000` | 频率超限报警 |
| LED2 | `F < FL && D > DH`，持续2秒 | 低流量高占空比异常 |
| LED3 | `F > FH && P < PL`，持续2秒 | 高流量低压力异常 |
| LED4 | D 正在变化中 | 占空比调节指示 |

LED2/LED3 的延时逻辑：当条件首次满足时，清零对应定时器 CNT，当 CNT > 20000（2秒）后才点亮 LED。条件不满足时立即熄灭。

### 6.4 控制算法

#### 6.4.1 手动模式

目标：使实际占空比 D 逐步趋近目标占空比 TarD。

```c
if ((D / 100.0) > duty)  D--;
if ((D / 100.0) < duty)  D++;
```

注意代码中 `duty = TarD / 100.0`，所以实际比较的是 `D/100.0` 和 `TarD/100.0`，即比较 D 和 TarD。由于 D 是整数，这里 `(D/100.0)` 作为浮点数比较，理论上可以直接比较 `D > TarD` 和 `D < TarD`。当前实现虽然冗余但功能正确。

#### 6.4.2 自动模式

目标：使实际压力 P 趋近目标压力 TarP。

```c
if (P > (TarP + 0.5))  if (D > 5 && D <= 95) D--;
if (P < (TarP - 0.5))  if (D >= 5 && D < 95) D++;
```

控制逻辑为**带死区的 Bang-Bang 控制**（开关控制）：
- 当压力高于目标 + 0.5 BAR 时，减小占空比
- 当压力低于目标 - 0.5 BAR 时，增加占空比
- 死区范围为 ±0.5 BAR，避免在目标值附近频繁调节

调节速率为每秒 ±1%，属于**缓慢渐变控制**。

## 7. 中断与时序分析

### 7.1 中断优先级配置

| 中断源 | 优先级 | 说明 |
|--------|--------|------|
| SysTick | 15 (最低) | HAL 时基 |
| TIM3_IRQn | 0 | 输入捕获 |
| TIM6_DAC_IRQn | 0 | 流量累计 |
| TIM7_IRQn | 0 | 占空比调节 |

所有外设中断优先级相同（都为 0），按中断向量表自然顺序仲裁。

### 7.2 时序参数汇总

| 任务 | 触发方式 | 周期 | 执行位置 |
|------|---------|------|---------|
| 按键扫描 | 主循环轮询 | 50ms | `key_scan()` |
| LCD 刷新 | 主循环轮询 | 100ms | `lcd_show()` |
| ADC 采集 | 主循环轮询 | 每次循环 | `adc_proc()` |
| PWM 更新 | 主循环轮询 | 每次循环 | `pwm_proc()` |
| LED 更新 | 主循环轮询 | 每次循环 | `led_proc()` |
| 频率测量 | TIM3 输入捕获中断 | 事件触发 | `HAL_TIM_IC_CaptureCallback` |
| 流量累计 | TIM6 定时中断 | 100ms | `HAL_TIM_PeriodElapsedCallback` |
| 占空比调节 | TIM7 定时中断 | 1s | `HAL_TIM_PeriodElapsedCallback` |

### 7.3 时序图

```
时间轴 ──────────────────────────────────────────────────────────>

主循环:  │ key │ adc │ pwm │ lcd │ led │ key │ adc │ pwm │ led │ key │ ...
         │ 50ms│     │     │100ms│     │ 50ms│     │     │     │ 50ms│
         └─────┘     └─────┘     └─────┘     └─────┘     └─────┘
```

## 10. 关键代码解析

### 10.1 PWM 占空比更新

```c
void pwm_proc(void)
{
    duty = TarD / 100.0;
    TIM2->CCR2 = (uint32_t)((D / 100.0) * (TIM2->ARR + 1));
}
```

这里 `duty` 被赋值为 `TarD/100.0`，但后续 CCR2 的计算使用的是 `D`（实时占空比），不是 `duty`。`duty` 变量在自动模式的 TIM7 中断中并未被使用，手动模式中虽然和 D 比较，但实际上不影响 CCR2 的值。CCR2 始终跟随 `D` 的变化。

### 10.2 自动模式控制逻辑

```c
if (htim->Instance == TIM7)
{
    if (work_mode == 0)  // 手动模式
    {
        if ((D / 100.0) > duty) D--;
        if ((D / 100.0) < duty) D++;
    }
    else  // 自动模式
    {
        if (P > (TarP + 0.5)) if (D > 5 && D <= 95) D--;
        if (P < (TarP - 0.5)) if (D >= 5 && D < 95) D++;
    }
    if (D_last == D) led4_mode = 0; else led4_mode = 1;
    D_last = D;
}
```

注意：自动模式的边界条件为 `D > 5 && D <= 95`（减）和 `D >= 5 && D < 95`（加），两侧不对称，但功能上不影响。

### 10.3 LED 延时点亮逻辑

```c
if (F < FL && D > DH)  // LED2 条件
{
    if (led2_first)      // 首次满足条件
    {
        TIM1->CNT = 0;   // 清零计时器
        led2_first = 0;
    }
    if (TIM1->CNT > 20000)  // 2秒后
    {
        led_show(2, 1);       // 点亮 LED2
    }
}
else
{
    led_show(2, 0);       // 立即熄灭
    led2_first = 1;
}
```

该逻辑实现了**条件持续满足 2 秒后才点亮**的功能，属于一种简单的延时触发机制。注意：如果条件满足不到 2 秒就解除，计时器会被清零，下次满足需重新计时。

## 11. 已知问题与改进建议

### 11.1 已知问题

1. **编码问题**: fun.c 中的中文注释在 UTF-8 编码下显示为乱码，原始编码可能为 GB2312/GBK。建议使用统一的 UTF-8 编码保存源文件。

2. **Q 累计公式**: 当前代码 `Q += (uint32_t)(F - 0.5)` 中，F 单位是 L/M（升/分钟），而中断周期是 100ms（即 1/600 分钟），理论上应乘以时间系数。当前实现会大幅高估累计流量。如果赛题要求如此则无需修改，否则应改为：
   ```c
   Q += (uint32_t)(F * 100 / 60);  // F(L/M) * 0.1s / 60s ≈ F/600
   ```

3. **手动模式 duty 未使用**: `pwm_proc()` 中计算 `duty = TarD/100.0`，但 `duty` 未用于更新 CCR2。实际手动模式下 D 由 TIM7 中断调节，功能正确但代码冗余。

4. **边界不对称**: 自动模式中 `D` 减的边界是 `D > 5 && D <= 95`，加的边界是 `D >= 5 && D < 95`，两侧不严格对称（一侧含等号一侧不含）。

5. **TIM1/TIM17 初始化**: CubeMX 中 TIM1 和 TIM17 仅配置了 Prescaler 和 Period，无其他功能配置。代码中直接操作 `TIM1->CNT` 和 `TIM17->CNT` 寄存器使用，这是可行的但需要确保定时器时钟已使能且计数器正在运行。

6. **D 的初始值**: `D` 初始化为 6，但 `TarD` 初始为 5，`D_last` 初始为 5。首次上电 TIM7 中断中 D 会从 6 递减到 5，led4_mode 会先置 1 再置 0。

### 11.2 改进建议

1. **使用浮点运算优化**: 流量累计和压力计算涉及浮点运算，在 80MHz Cortex-M4 上性能足够，但可考虑定点化优化。

2. **增加参数存储**: 当前参数（FH/FL/PL/DH/TarD/TarP）为 RAM 变量，掉电丢失。可扩展 Flash 模拟 EEPROM 保存参数。

3. **控制算法升级**: 当前自动模式为 Bang-Bang 控制，可升级为 PID 控制以实现更平滑的压力调节。

4. **界面反显**: 当前 OUTP/PARA 界面没有显示当前选中项的反显（高亮）效果，可通过 `LCD_SetBackColor` / `LCD_SetTextColor` 交换实现选中项反显。

5. **代码规范化**: 
   - 统一变量命名风格（当前混合了驼峰和下划线）
   - 将魔法数字提取为宏定义
   - 添加必要的边界检查

### 12.3 ADC 电压换算

```
ADC_Value (0~4095) → Voltage (0~3.3V):
    Voltage = 3.3 * ADC_Value / 4095.0

Voltage → Pressure (0~10 BAR):
    if (Voltage >= v_offset):
        P = (Voltage - v_offset) / (3.3 - v_offset) * 10.0
    else:
        P = 0
```

## 附录 B：FreeRTOS 多任务改造技术总结

> **改造日期**: 2026-06-21  
> **原始架构**: 裸机 while(1) 超级循环 + 定时器中断  
> **改造后架构**: FreeRTOS 多任务 + 互斥量 + 二值信号量  
> **目标**: 将 FreeRTOS 从"了解"提升到"熟练使用"，支持简历和面试

### B.1 改造动机

原裸机代码虽然功能完整，但存在以下局限：
- **轮询式延时**（`HAL_GetTick()` 判断周期）占用 CPU，实时性差
- **中断中直接执行业务逻辑**（TIM6/TIM7 中做 Q 累计、PWM 调节），ISR 执行时间长
- **全局变量无保护**，裸机下虽不出错但不符合工程规范
- **功能扩展困难**：加新功能要改主循环时序，耦合度高

### B.2 架构改造对比

| 维度 | 裸机版本 | FreeRTOS 版本 |
|------|---------|-------------|
| 主循环 | `while(1)` 轮询 6 个函数 | 6 个独立任务，调度器管理 |
| 延时方式 | `HAL_GetTick()` 轮询判断 | `vTaskDelayUntil()` 精确周期 |
| 定时中断 | ISR 中直接执行业务逻辑 | ISR 只发信号量，任务中执行业务 |
| 数据同步 | 无保护（全局变量裸奔） | 互斥量（Mutex）保护共享数据 |
| 优先级 | 无（顺序执行） | 6 级抢占式优先级 |
| 扩展性 | 差（改主循环时序） | 好（加任务不影响现有任务） |

### B.3 任务设计

```
┌─────────────────────────────────────────────────────────────┐
│                     FreeRTOS 抢占式调度器                      │
├─────────────┬─────────────┬─────────────┬───────────────────┤
│ Task_Sensor │ Task_Control│ Task_Input  │   Task_Display    │
│ 优先级: 3   │  优先级: 4  │ 优先级: 3   │    优先级: 2      │
│ 周期: 50ms  │ 触发: 信号量 │ 周期: 50ms  │    周期: 100ms    │
│ ADC采集     │  PWM调节    │ 按键扫描    │    LCD刷新        │
│ 压力/流量   │  Q累计      │ 参数修改    │                   │
└─────────────┴─────────────┴─────────────┴───────────────────┘
├─────────────────────┬─────────────────────┐
│  Task_FlowAccum     │    Task_LED         │
│  优先级: 3          │    优先级: 1        │
│  触发: TIM6信号量   │    周期: 100ms      │
│  Q累计              │    LED刷新          │
└─────────────────────┴─────────────────────┘
```

**优先级设计理由**：
- **PWMControl (4)**: 闭环控制实时性要求最高
- **Sensor/Flow/Key (3)**: 数据采集和输入，需要及时响应
- **Display (2)**: LCD 刷新允许轻微延迟
- **LED (1)**: 状态指示最不敏感

### B.4 同步机制

| 对象 | 类型 | 用途 | 使用位置 |
|------|------|------|---------|
| xSensorDataMutex | 互斥量 | 保护 P, F, r37_volt | Sensor(写) / Display/PWM/LED(读) |
| xControlDataMutex | 互斥量 | 保护 D, TarD, TarP, Q, work_mode | PWM/FlowAccum/Key/Display/LED |
| xLcdMutex | 互斥量 | 保护 LCD 硬件 | Display(写) / Key(清屏) |
| xTim6Semaphore | 二值信号量 | TIM6 → FlowAccum | TIM6 ISR 发，FlowAccum 收 |
| xTim7Semaphore | 二值信号量 | TIM7 → PWMControl | TIM7 ISR 发，PWMControl 收 |

**死锁防护**：所有任务遵循统一加锁顺序：**先 SensorData → 后 ControlData → 再 LCD**。

### B.5 ISR 精简设计（RTOS 黄金法则）

```c
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    BaseType_t xWoken = pdFALSE;
    if (htim->Instance == TIM6)
        xSemaphoreGiveFromISR(xTim6Semaphore, &xWoken);
    else if (htim->Instance == TIM7)
        xSemaphoreGiveFromISR(xTim7Semaphore, &xWoken);
    portYIELD_FROM_ISR(xWoken);
}
```

**设计原则**：ISR 尽可能短（微秒级），复杂计算放到任务中做（毫秒级）。

### B.9 CubeMX 注意事项

1. **警告"use a HAL timebase source other than Systick"**：点 Yes 继续，不改配置（标准做法）
